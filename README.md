# event-broker-api
RESTful API сервис для управления мероприятиями (событиями) с полным CRUD функционалом.

## 📋 Описание
API позволяет создавать, читать, обновлять и удалять мероприятия. Также формировать бронь на мероприятие, читать и обрабатывать ее. Каждое событие содержит информацию о названии, времени проведения, описании и информацию о бронировании на него.

## 🧰 Функциональность
| Метод | URL | Описание | Payload |
|-------|-------|-------|-------|
| **GET**         | /events |получение списка всех событий           |-|
| **GET**         | /events/{eventId:guid}/bookings/{bookingId:guid} |получение информации о бронировании на событие           |-|
||*Параметры фильтрации*||
| **GET/{id}**    | /events/{id:guid} |получение события по идентификатору |-|
| **POST**        | /events |создание нового события |json модель события (см ниже)|
| **POST**        | /events/{eventId:guid}/book |создание неподтвержденного бронирования на мероприятие |json модель бронирования (см ниже)|
| **PUT/{id}**    | /events/{id:guid} |полное обновление существующего события |json модель события (см ниже)|
| **DELETE/{id}** | /events/{id:guid} |удаление события |-|

**примечание** 

Теперь вызов метода создания бронирования **POST** /events/{eventId:guid}/book связан с кол-вом свободных мест.
Если очередной запрос брони на мероприятие превысит это кол-во вернется ответ **409 Conflict**

### Параметры фильтрации метода GET
|параметр|тип|описание |примечание|
|-------|-------|-------|-------|
|Title|string|Фильтр по названию события|частичное совпадение, не зависит от регистра|
|From|string|Начало события|включительно (формат ISODateString)|
|To|string|Конец события|включительно(формат ISODateString)|
|ValidatePeriod|boolean|Включить проверку|что начало раньше завершения события,|
|Page|int|Номер страницы| по умолчанию 1|
|PageSize|int|Количество записей на странице|по умолчанию 10|
|OrderBy|string|Сортировка результатов| не реализовано|

***пример запроса события***

**GET /events**?Title=конференция&From=2026-05-01T00:00:00Z&To=2026-05-31T23:59:59Z&ValidatePeriod=true&CheckPeriod=false&Page=1&PageSize=20&OrderBy=date%20desc

***пример бронирования на мероприятие***

**Сервис обработки бронирования** - отдельный фоновый процесс, ускоряющий UX работы с API

use case прост (с возможным овербукингом):
1. выполнив запрос на создание события **POST /events**, вы получите 201 c location, содержащим уникальный id (guid) только что созданного события
2. выполните запрос на создание неподтвержденной брони на вышеуказанное событие **POST /events/{eventId:guid}/book**
2.1 если очередной запрос брони на мероприятие превысит это кол-во доступных мест вернется ответ **409 Conflict**
3. получив на клиенте 202 ответ, что бронь сформирована, вы также получите location на созданную бронь
4. периодически опрашивайте **GET /bookings/{bookingId:guid}** API о созданной брони, чтобы получить более подробную информацию о ее текущем статусе (что она подтверждена или отменена)

** примечание **

Применены примитивы синхронизации для защиты от непредсказуемых результатов и обеспечения целостности данных при состоянии гонки или попытки оверукинга:
1. *lock (_lockObject)* - гарантирует, что только один поток одновременно может находиться внутри защищенного блока кода и работать с переменными в нем. **Нельзя использовать с async/await**
2. *SemaphoreSlim* - ограничение количества одновременно выполняемых асинхронных операций. Метод WaitAsync не блокирует поток — позволяет эффективно ждать в асинхронном коде.

## 📦 Модели данных

## Event (Событие)

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| Id | Guid | Да | Уникальный идентификатор события |
| Title | string | Да | Наименование мероприятия |
| StartAt | DateTime (ISO формат) | Да | Дата и время начала |
| EndAt | DateTime (ISO формат) | Да | Дата и время завершения |
| Description | string | Нет | Описание мероприятия |
| TotalSeats | int | да | Общее количество мест на мероприятии |
| AvailableSeats | int | Нет | Количество оставшихся свободных мест на мероприятии (0 <= AvailableSeats <= TotalSeats) |

## Booking (Бронирование на событие)

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| Id | Guid | Да | Уникальный идентификатор события |
| EventId | Guid | Да | Идентификатор мероприятия |
| Status | BookingStatus (см ниже) | Да | Статус бронирования |
| CreatedAt | DateTime (ISO формат) | Да | Дата и время бронирования |
| ProcessedAt | DateTime (ISO формат) | Нет | Дата и время обработки брони (смены статуса)|

## BookingStatus (enum - Статусы бронирование на событие)
| Поле | Значение | Описание |
|------|----------|----------|
| Pending | 0 | Бронь создана, но не подтверждена |
| Confirmed | 1 | Бронь подтверждена |
| Status | 2 | Бронь отклонена |


### 🚀 Установка и запуск
1. откройте терминал
2. клонируйте проект **git clone -b sprint_4 https://github.com/SerikovGennadiy/event-broker-api.git**
3. перейдите в директорию **cd event-broker-api (содержащую sln файл решения)**
4. запустите тесты **dotnet test ./EventBrokerAPI.Tests/EventBrokerAPI.Tests.csproj**
5. убедитесь, что порт указанный в настройках запуска свободен
6. выполните **dotnet run --project ./EventBrokerAPI/EventBrokerAPI.csproj**
